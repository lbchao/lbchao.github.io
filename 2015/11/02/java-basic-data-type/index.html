java_basic_data_type

java基本数据类型及相关介绍。

java是强类型语言，强类型包含两方面含义：
1：所有的变量必须先声明，后使用；
2：指定类型的变量只能接受类型与之匹配的值。
这意味着每个变量和每个表达式都有一个在编译时就确定的类型，类型限制了一个变量能被赋的值，限制了一个表达式可以产生的值，限制了在这些值上可以进行的操作，并确定了这些操作的含义。
强类型语言可以在编译时进行更严格的语法检查，从而减少编程错误。

java 支持的类型分为两类：基本类型 和引用类型

基本类型包含boolean类型和数值类型。数值类型有整数类型和浮点类型。整数类型包含byte，short,int ,long,char,浮点类型包含float和double。
（char代表字符型，实际上字符型也是一种整数类型，相当于无符号整数类型）
引用类型包含类，接口，和数组类型，
null, 空类型，null是null类型变量唯一的值，空引用可以转换为任何引用类型，
（空引用（null）只能被转换成引用类型，不能转换成基本类型，因此不要把一个null值赋给基本数据类型的变量）。

基本数据类型 整数类型 byte (一个字节)
short （两个字节）
int （4个字节）
long （8 个字节）

                 字符类型   char   （2 个字节）

                 浮点类型   float   （4个字节） 
                            double  （8 个字节） 

                 布尔类型 boolean

byte  一个byte类型整数在内存里占8位，范围是-128（-2的7次幂)~-127(2的7次幂-1)
short 一个short类型整数在内存里占16位，-32768（-2的15次幂）~-32767（2的15次幂-1）
int   一个int类型整数在内存里占32位，表示范围：-2147483548（-2的31次幂）~（2的31次幂-1）
long  一个long类型整数在内存里占8位，范围：（-2的63次幂）~（2的63次幂-1）

int是常用的整数类型，通常情况下，直接给一个整数，默认是int类型，除此之外，有以下两种情况：

1 如果直接将一个较小的整数值（在byte，short 范围）赋给一个byte或者short变量，系统会自动把这个整数值当成byte或者short类型处理。
2如果使用一个巨大的整数值（超过int范围）java不会自动把这个整数值当初long来处理，如果希望系统把一个整数值当初long处理，应该在这个整数值后增加l或者L作为后缀。


（可以把一个较小的整数值（int范围内），直接赋值给一个long类型变量，并不是因为java会把这个较小的整数值当初long类型处理，java依然把这个整数值当int类型来处理，只是因为int类型的值会自动类型转换到long类型）

Java中整数值有4种表示方式： 十进制，二进制，八进制和十六进制。

十进制：即我们平常使用的计算方式
二进制：整数以0b或者0B开头
八进制：整数以0开头
十六进制整数0x或者0X开头， 10-15 分别以a-f(不区分大小写)来表示。

从上边最后一个值看，定义32位的二进制整数时，最高位其实是符号位，符号位是1时，表明它是一个负数，负数在计算机里是以补码的形式存在的，因此需要换算成原码。

所有数字在计算机底层都是以二进制形式存在的，原码是直接将一个数值换算成二进制数。
计算机以补码的形式保存所有的整数，补码的计算规则：
正数的补码和原码完全相同
负数的补码是其反码加1，反码是对原码按位取反，只是最高位（符号位保持不变。）

补码：0B 1000 0000 0000 0000 0000 0000 0000 0011
（反码=补码-1）
反码：0B 1000 0000 0000 0000 0000 0000 0000 0010

原码：0B 1111 1111 1111 1111 1111 1111 1111 1101（原码值）
（-2147483645）

整数值默认int类型，因此使用二进制定义整数时，二进制整数默认占32位，第32位是符号位；
二进制整肃后天就l或者L后缀，二进制整数默认占64位，其中，第64位是符号位。

字符型

字符类型通常用于表示单个字符，字符类型值必须使用单引号括起来。java语言使用16位的Unicode字符集作为编码方式，而Unicode被设计成支持世界上所有的书面语言的字符，包括中文字符，因此，java程序支持各种语言的字符。

浮点型
java的浮点类型有两种：float和double。double类型代表双精度浮点数，float类型代表单精度浮点数，
float 占4个字节，32位
double 占8个字节， 64位。

java语言的浮点数两种表示形式：
十进制数形式：这种形式就是简单的浮点数，例如 5.12,512.0，浮点数必须包含一个小数点，否则会被当成int类型处理。
科学计数法形式：例如5.12e2/5.12E2(5.121010)

必须指出，只有浮点类型的数值才可以使用科学计数法形式表示。
51200 int类型
512E2 浮点类型

java语言的浮点类型默认是double类型，如果希望java把一个浮点类型值当成float类型处理，应该在这个浮点类型后紧跟F或者f.
5.12 double 占64位
5.12f/5.12F float 占32位

当然，也可以在一个浮点数后天就D或者d后缀，强制指定double类型，但是通常没有必要。

java提供三个特殊的浮点数值：正无穷大，负无穷大，和非数。

必须指出的是，所有的正无穷大数值都是相等的，所有的负无穷大数值都是相等的，NAN不与任何数值相等。

java7引入了新功能，在数值中科院使用下滑线，不管是整型，浮点型，都可以自由的使用下画线，使用下画线分隔，可以更直观分辨数值到底包含多少位。

布尔型

布尔型只有一个boolean类型，用于表示逻辑上的“真”“假”，值true.false。其他基本数据类型的值不能转换成boolean类型。
java没有指定boolean类型的变量所占用的内存空间，虽然boolean类型的变量只要一位即可保存，但是由于大部分计算机在分配内存时允许分配的最小内存单元是字节（8位），因此大部分时候实际上占用8位。

基本数据类型的类型转换

java所有的数值型变量可以相互转换，如果系统支持把某种基本类型的值之间赋给另一种基本类型的变量，则这种方式
自动类型转换，
当吧一个范围小的数值或变量之间赋给另一个表示范围大的变量时，系统进行自动类型转换，否则需要强制转换。

%求余运算符，求余运算结果不一定总是整数，它的计算结果是使用第一个数除以第二个数，得到一个整除的结果后剩下的值就是余数。进行除法，两个数都是整数类型，第二个不能为0，如果有一个为浮点数，允许第二个为0.只是结果是非数：NaN。 0或者0.0对0以外的任何数求余都得到0或者0.0。

++ 自加

右边：先把操作数放入表达式中运算，才把操作数加1，

~：按位非。单目运算符，将操作数每个位（包括符号位）全部取反
^:按位亦或。当两位相同时返回0，不同时返回1.
<<:左移运算符
