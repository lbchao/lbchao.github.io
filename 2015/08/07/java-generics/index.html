java_generics

java5以后，java引入了“参数化类型”的概念，允许我们在创建集合时指定集合元素的类型，例如list,java的参数化类型称为泛型（Generic）.

List :集合接口，类后增加尖括号，尖括号里放一个数据类型，集合接口只能保存特定类型的对象。（list带类型参数的泛型接口）

java7之后就只需要一个尖括号就可以了，简化了写法。

泛型的含义：

泛型，在定义类，接口，方法时使用类型参数，这个类型形参在声明变量，创建对象，调用方法时动态的指定（即传入实际的类型参数，类型实参）。
java5改写了集合框架中的全部接口和类，为这些接口，类增加了泛型支持，从而可以在声明集合变量，创建对象时传入类型实参， 例如：list ,ArrayList。

//允许在类接口里声明类型形参，类型形参在整个接口，类体内可以当成类型使用。

总结：创建了带泛型的接口，父类后，可以为接口创建实现类，或者从父类派生子类，注意，使用这些接口，父类时不能再包含类型形参。如上：

并不存在泛型类

java把他们当成一个类处理，在内存中占用一块内存空间，因此在静态方法，静态代码块，或者静态变量中和初始化，不允许使用类型形参。

泛型方法

，定义类和接口使用类型形参，在类中方法定义，字段定义，接口方法定义，这些类型形参可被当成普通类型来用，在另外一些情况下，我们定义类，接口时没有使用类型形参，但是定义方法时也可以定义类型形参，java5提供了对泛型方法的支持。
泛型的格式：
修饰符 返回值类型 方法名（形参列表）

形参放在修饰符和返回值之间

1泛型方法和类型通配符的区别

看下Collection类中的方法

通配符设计，让子类设计变的灵活，
泛型方法允许类型形参被用来表示方法的一个参数或多个参数的类型依赖关系，方法返回值和参数之间类型依赖关系，没有这种依赖关系，就不应该使用泛型方法。

两者还有一个区别：
类型通配符即可以在方法签名中定义形参的类型，定义变量的类型;泛型方法的形参必须对应方法中显示声明。

泛型构造器

泛型方法允许在方法签名中声明类型形参，java允许在构造器签名中声明类型形参，这就是所谓的泛型构造器。

java7菱形用法，允许调用构造器时在构造器后使用一对尖括号来代表泛型信息。但是如果程序显示指定了泛型构造器中声明的
类型形参的实际类型，则不可以使用菱形语法。

泛型和数组

java泛型的设计原则：如果一段代码在编译时没有提出“未经检查的转换”警告，程序运行时不会引发ClassCastException异常。
数组元素类型不能包含类型变量或者类型形参，除非是无上限的类型通配符。
可以声明类型包含类型变量或者类型形参的数组。只能声明List[],不能创建ArrayList[10]这样的数组对象
java 允许创建无上限的通配符泛型数组 new ArrayList<?>[10]
